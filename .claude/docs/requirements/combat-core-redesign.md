## 0. 背景と目的

現行戦闘は「毎ターン energy 全回復・固定 5 ドロー・速度はフェーズ順のみ」で、選択肢はほぼ一定・火力は固定。これを **「現実的な戦闘の手触り」** ＝ *開幕が最も鋭く、消耗して泥仕合になる* 戦闘へ**大幅改修**する。タイマー（実時間の焦り）と合わせ、戦闘コアを一括で引き直す。

**狙いの体験**: スタミナ（疲労）が攻防と手数のレバーになり、「疲れる前に押し切る／相手を消耗させて仕留める」という決断ゲームにする。

## 1. 確定した設計判断

### 1.1 本改定（2026-06-08 ユーザー回答）

| 論点 | 確定 | 含意 |
| --- | --- | --- |
| **スタミナ構造** | **単一通貨 + 下限保証** | 行動・火力・防御を 1 つのスタミナが担う。ただし最低火力/防御/行動を保証しデススパイラルを緩和 |
| **戦闘の長さ** | **短期決戦 3〜6 ターン** | 速攻で勝つ/長引けば不利を明確な軸に。敵 HP・報酬・回復設計が連動 |

最小テーマ改定 / タイムアウト=強制ターン終了 / ターン全体の単一カウントダウン・常時進行 / 持ち時間は 4 要因合成。

## 3. 中核モデル

### 3.1 スタミナ（単一通貨 + 下限保証）
- **担う役割**: ①カードを切る行動コスト ②そのターンの**攻撃 tier 倍率** ③そのターンの**防御 tier 倍率**。
- **ターン 1 は満タン始動**（front-loaded＝開幕が最も鋭い）。
- **毎ターン部分回復**（全回復しない）→ 長引くほど total が下がる＝**疲労**。
- **下限保証**: スタミナが尽きても (a) 攻撃 tier は最低 ×0.8 (b) 最低防御を維持 (c) 最低 1 行動は可能。→ 全次元同時崩壊（デススパイラル）を防止。

### 3.2 tier（ターン開始時に確定し、当該ターン固定）
| tier | スタミナ残量 | 攻撃倍率 | 被ダメージ倍率（防御） |
| --- | --- | --- | --- |
| 絶好調 Fresh | ≥ 70% | ×1.2 | ×0.85 |
| 平常 Normal | 30–69% | ×1.0 | ×1.0 |
| 消耗 Exhausted（=下限） | < 30% | ×0.8 | ×1.15（下限保証で頭打ち） |

> **要点**: tier は**ターン開始時のスタミナ**で決まり、ターン中にカードを切ってスタミナが減っても**当該ターンの火力 tier は変わらない**。これにより「1 ターン内で 2 撃目以降が弱くなる」破綻を構造的に回避する。ターン中のスタミナ消費は純粋に**行動経済（手数）**のみを縛る。

### 3.3 ドロー曲線（手数は逆に増える）
- ターン 1 は少（例 3 枚）→ 数ターンで上限（例 5 枚）。「序盤は準備不足で手数が少ない」を表現。
- バフ/魔法で増、敵デバフで減。**手札破壊は特殊敵限定**（常設にしない）。
- 結果として **序盤＝少ない手数で重い一撃 / 終盤＝多い手数で軽い手数** という対比が生まれる（疲労 × 手数増の意図的な綱引き）。

### 3.4 速度 → 火力
- フェーズキューで相手より先に動けたら**初撃 +X%**（既定 +10%）。連続値の複雑な式は入れず「先手＝有利」の単調ボーナスのみ。

### 3.5 スタミナ戦（勝ち筋の可視化）
- 一部カード/敵行動で**相手スタミナを削る** → 相手を早く「消耗」tier に落とし、火力・手数・防御を同時に下げる。「消耗させて仕留める」が明示的戦術になる。
- 逆に魔法/バフで自分のスタミナを回復・増加。

### 3.6 既存 `ap` / `guard` の統合（推奨案・要確定）
- **`guard`**: 従来どおり、カードで張る**能動的な一時防御**として残す。
- **`ap`（緩衝層）**: **下限保証＝削られない最低靭性**として転用。スタミナが尽きても一定の被ダメージ緩衝を残し、消耗 tier での頭打ちを担保。
- スタミナ単一通貨は「行動＋火力＋防御倍率」、`guard` は「能動ブロック」、`ap` は「最低靭性」と役割分離。

## 4. 機能要件（FR）

- **FR-1 スタミナ単一通貨**: 行動コスト・攻撃 tier・防御 tier を 1 プールで管理。
- **FR-2 下限保証**: 0 スタミナでも最低火力(×0.8)・最低防御・最低 1 行動を保証。
- **FR-3 tier 確定**: 攻撃/防御倍率はターン開始時スタミナで確定し当該ターン固定。
- **FR-4 部分回復/疲労**: ターン 1 満タン、以降は部分回復で total が逓減。
- **FR-5 ドロー曲線**: 序盤少→上限へ。バフ増/デバフ減。手札破壊は特殊敵限定。
- **FR-6 速度ボーナス**: 先手時に初撃 +X%。
- **FR-7 スタミナ戦**: 相手スタミナを削る/自分を回復する効果をカード・バフ・敵行動に持たせる。
- **FR-8 短期決戦バランス**: 敵 HP・報酬・回復を 3〜6 ターン決着前提で設計。
- **FR-9 タイマー統合**: `realtime-turn-timer.md` の常時進行タイマーと両立（スタミナ＝物語内 / タイマー＝物語外、二重罰にしない）。

## 5. 非機能要件（NFR）

| ID | 要件 |
| --- | --- |
| **NFR-1 暗算可能性** | tier は 3 段・ターン開始時確定。リアルタイム下でも結果を予測できる。各倍率は UI で明示。 |
| **NFR-2 デススパイラル防止** | 下限保証により劣勢でも反撃の芽を残す。検証で「劣勢からの逆転率」を観測する。 |
| **NFR-3 二重罰の回避** | タイマーはスタミナを削らない。両者は独立した圧力軸。 |
| **NFR-4 既存規約** | React 19 パターン・状態単一所有・不可侵コード（`deck.ts`/`deckReducer.ts`）厳守。`energy` 改名は型/UI/テストへ波及するため一括で。 |
| **NFR-5 決定性** | スタミナ/tier/ドロー数は装備・深度・難易度から決定的に再現可能。 |

## 7. アーキテクチャ影響

> 既存: `useBattleState`（HP/AP/Guard/Energy/Buffs）、`damageCalculation`（baseDamage×攻撃バフ×防御）、`playerPhaseExecution`（毎ターン energy 全回復・固定 5 ドロー）、`speedCalculation`（順序のみ）。

- **`energy` → `stamina` 再定義/改名**: 型(`battleTypes`/`characterTypes`)・`useBattleState`・`useCardExecution`・UI(`PlayerFrame` 等)・テストへ波及。一括改名。
- **tier 計算**: `domain/battles/calculators/` に純粋関数 `calculateStaminaTier(stamina, maxStamina)` を新設。`damageCalculation` に tier 倍率（攻撃/被ダメージ）を組み込む。tier は**ターン開始時**に確定し state に保持。
- **部分回復**: `executeCharacterManage.ts` の「毎ターン energy 全回復」(現 `setPlayerEnergy(maxEnergy)`) を**部分回復＋最低保証**へ置換。
- **ドロー曲線**: `PLAYER_BASE_DRAW_COUNT`（固定 5）を**ターン数依存の曲線**へ。`calculatePlayerPhaseStart` に turn 引数を渡す。
- **速度ボーナス**: `damageCalculation` に「先手判定 → 初撃倍率」を追加（フェーズキュー結果を参照）。
- **下限保証/AP 転用**: `applyDamageAllocation`（Guard→AP→HP）の AP を「最低靭性」として再解釈。
- **スタミナ戦**: バフ種に stamina 増減を追加（`battle-system` スキル）、カード/敵行動データを追加（`card-creator`/`enemy-creator`）。
- **タイマー統合**: `useTurnTimer`（タイマー要件）と本モデルを `useBattleOrchestrator` 配下で統合。

## 8. 受け入れ基準（Iteration 0 / MVP）

- **Given** ターン 1 **When** 戦闘開始 **Then** スタミナ満タン＝**絶好調 tier（×1.2）**で、手札は少ない（例 3 枚）。
- **Given** ターンが進む **When** 部分回復しか起きない **Then** スタミナ total が逓減し、数ターンで**消耗 tier（×0.8）**へ移行する。
- **Given** ターン中にカードを複数枚切る **When** スタミナが減る **Then** **当該ターンの火力 tier は変わらない**（ターン開始時で固定）。
- **Given** スタミナ 0 **When** 行動 **Then** 最低火力(×0.8)・最低防御・最低 1 行動が保証される（全崩壊しない）。
- **Given** 相手スタミナを削るカード **When** 使用 **Then** 相手が早期に消耗 tier へ落ちる。
- 既存テストが全て pass（回帰ゼロ）。

## 10. タイマー要件との統合関係

- **役割分担**: スタミナ＝物語内の消耗（疲労・削り合い）、タイマー＝物語外の焦り（実時間）。**タイマーはスタミナを削らない**（NFR-3）。
- **相乗**: tier がターン開始時に確定するため、プレイヤーは「このターンの power」を把握した上で、10 秒の実行プレッシャーに臨める（計画は事前、実行は即断）。
- **持ち時間 4 要因**との接続: スタミナ/速度ビルドは持ち時間にも寄与しうる（`realtime-turn-timer.md` §6 の playerTempoBonus と整合させる）。

## 11. 未決事項（次イテレーションで確認）

1. **手札破壊の範囲**: 特殊敵限定で確定か（推奨）。
2. **タイマー独立**: タイマーはスタミナを削らない＝独立軸で確定か（推奨）。
3. **`ap` 転用**: `ap` を「最低靭性」に転用する案（§3.6）の可否。緩衝層を残すか畳むか。
4. **tier 閾値/倍率**・**回復率/下限**・**ドロー曲線**・**速度ボーナス量**の数値（要プレイテスト）。
5. **ターン開始時 tier 確定** vs 「行動直前 tier」: 暗算性は前者が上。前者で確定か。
6. **試験バトル**（`GuildBattleScreen`）への適用可否。
7. **マルチ敵**時のスタミナ戦の扱い。

## 12. リスクと対策

| リスク | 影響 | 対策 |
| --- | --- | --- |
| 下限保証が弱く結局デススパイラル | 逆転不能で敗者が見てるだけ | NFR-2 で逆転率を観測。下限値を調整可能な定数に |
| 逆転曲線が「やるほど悪化」で陰鬱 | モチベ低下 | 短期決戦(3〜6T)で「速攻で勝つ」快感に寄せる。C7 で体感調整 |
| `energy`→`stamina` 改名の波及漏れ | 型/UI/テスト破壊 | C1 を独立スライスにし回帰ゼロをゲート（`card-battle-battle-logic-validator`） |
| tier × タイマー × 装備の調整爆発 | N=1 で破綻 | 結合を 1 スライスずつ導入。各々単体テスト可能な純粋関数に |
| 常時進行タイマー × 重いアニメ | 理不尽化 | `realtime-turn-timer.md` §7.5/§12（アニメ非ブロッキング・PixiJS Phase 2 連動） |
