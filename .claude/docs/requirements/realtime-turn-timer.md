## 0. 背景と目的

### 0.1 テーマ改定

現行テーマ「風来のシレン的エクストラクション」の**参照を外し**、ゲームのアイデンティティを **「本格的な戦略 × リアルタイム緊張のカードバトル」** へ転換する。

- **維持（最小改定）**: エクストラクション構造（持ち込み/持ち帰りリスク）・ライフ制・ソウル 100% 獲得は**そのまま残す**。
- **撤去**: 「シレン」という参照タイトルのみ（構造は残す）。
- **追加（根幹）**: プレイヤーのターンに**リアルタイムの持ち時間**を導入する。

### 0.2 中核要求

> プレイヤーのターンに持ち時間（**約 10 秒・バトルごとに変動**）を設け、その制限内に「どのカード（＝どの行動）を取るか」を選択させる。

これによりゲームは「速度将棋（speed chess）」的な緊張感を持ち、熟考ではなく**事前ビルド＋即断**で勝つゲームになる。

## 2. スコープ

### 2.1 In（本要件で扱う）

- プレイヤーターンの持ち時間カウントダウン（単一・常時進行）。
- タイムアウト → 強制ターン終了。
- 持ち時間の算出モデル（4 要因合成）。
- 時間を増減させるカード/バフ効果（「時間」をゲーム内リソース化）。
- タイマー UI（可視カウントダウン＋緊張演出）。
- 難易度連動（`TIME_BY_DIFFICULTY`）。
- 通常バトル（`BattleScreen`）への適用。

### 2.2 Out（本要件では扱わない / 別途）

- エクストラクション/ライフ制ロジック自体の変更（維持）。
- 敵フェーズの時間制限（敵は従来どおり AI 自動進行・**非計測**）。
- ギルド昇格試験バトル（`GuildBattleScreen`）への適用可否 → §11 で要判断。
- リアルタイム化に伴う敵 AI のリアルタイム挙動（敵は引き続きターン制）。
- 具体的な数値バランス確定（§6 は初版・要チューニング）。

## 3. 設計上の中心的緊張と解決方針

**緊張**: 「深い戦略」は熟考時間を要するが、「常時進行の 10 秒制限」は熟考を罰する。両者は構造的に綱引きする。

**解決方針（front-loaded strategy）**:

1. **戦略は事前に寄せる** — デッキ構築・装備・速度ビルド・熟練度といった**バトル前の意思決定**に戦略性を集約する。バトル中は「組んできた手を即座に正しく切る」実行ゲームとする。
2. **時間そのものを戦略変数にする** — 速度/装備で持ち時間を伸ばし、カードで時間を増減させる（時間 = リソース）。「速度ビルド vs 火力ビルド」のトレードオフを生む。
3. **常時進行ゆえアニメは演出であって障害物にしない** — カード解決アニメは**ノンブロッキング**（タイマー・入力を止めない）に限定する。これは技術的な必須条件（§7.5）。
4. **難易度で緩急** — Easy は熟考余地を残し、Hard は反射神経も要求する。

## 4. 機能要件（Functional Requirements）

### FR-1 タイマーの存在と範囲
プレイヤーの**ターン全体**に対し、**単一のカウントダウン**を持つ。ターン開始時にそのターンの持ち時間 `turnTimeSec` がセットされ、0 へ向けて減少する。

### FR-2 常時進行（停止しない）
カウントダウンは**プレイヤーの意思決定ウィンドウ中、常に進行**する。カード選択・カード解決・ダメージ/バフ適用・各種アニメーション中も**停止しない**。
- ただし**敵フェーズ・フェーズ遷移・ターン間メッセージ表示中は計測しない**（プレイヤーが操作できない時間は持ち時間を消費させない）。
- すなわちタイマーが動くのは「プレイヤーが行動を選べる実時間」のみ。

### FR-3 タイムアウト = 強制ターン終了
`turnTimeSec` が 0 に達したら、**そのターンを即座に強制終了**する。
- 未使用 AP・残り手札は**放棄**（次ターンへ繰り越さない。既存の手札破棄/ターン終了処理に合流）。
- 解決中のカードがあれば、その 1 枚は最後まで解決してから終了する（不整合防止。詳細挙動は §11 で要確定）。

### FR-4 持ち時間の算出（4 要因合成）
`turnTimeSec` は次の 4 要因を合成して決まる（モデルは §6）:
1. **難易度基準値**（`TIME_BY_DIFFICULTY`）
2. **敵/深度補正**（強敵・高深度ほど短縮）
3. **プレイヤー速度/装備補正**（`speed` ステータス・装備で延長）
4. **カード/効果補正**（時間バフ・デバフ・即時加算）

### FR-5 カード/効果による時間増減（時間 = リソース）
カード効果・バフ/デバフで持ち時間を増減できる。
- **持続型**（バフ）: 以降のターンの `turnTimeSec` を ±X 秒する。
- **即時型**: 現在進行中のカウントダウンに +X 秒（時間の「回復」）。
- 敵側のデバフとして「プレイヤーの持ち時間短縮」を持てる（緊張演出）。

### FR-6 早期終了
プレイヤーは時間内でも能動的にターンを終了できる（既存の「ターン終了」操作）。AP 枯渇/手札なしでも終了可能。AP 枯渇時に自動終了するかは §11 で要確定。

### FR-7 タイマー UI（緊張演出）
- 残り時間を**バー＋数値**で常時可視化（`ui-ux-creator` スキル準拠、ダークファンタジー調）。
- 残量に応じた色変化（緑 → 橙 → 赤）と終盤の鼓動/パルス演出。
- 「持ち時間が変動した」フィードバック（バフ/装備で延びた等）。
- 配置は `.battle-screen` 内にスコープ（CSS 親スコープ規約）。

### FR-8 難易度連動
`LIVES_BY_DIFFICULTY` と同様に `TIME_BY_DIFFICULTY`（Easy/Normal/Hard）を定義し、基準値を切り替える。

## 5. 非機能要件（Non-Functional Requirements）

| ID | 要件 |
| --- | --- |
| **NFR-1 精度** | カウントダウンは実時間基準（`performance.now()` + `requestAnimationFrame`）。`setInterval` の累積ドリフトを避ける。表示更新は ~10fps で十分だが内部計測は連続値。 |
| **NFR-2 ノンブロッキング** | カード解決・アニメが**メインスレッドや入力を長時間ブロックしない**こと（§7.5）。常時進行ゆえブロッキングは即「理不尽」になる。 |
| **NFR-3 公平性** | タイムアウト直前の入力に**最小猶予**（例: 入力確定の取りこぼし防止デバウンス）。タブ非アクティブ/復帰時の挙動を定義（一時停止 or そのまま）。 |
| **NFR-4 アクセシビリティ** | 反射速度要求はアクセシビリティ障壁になりうる。設定で「タイマー延長/無効化（練習モード）」トグルの提供を検討（§11）。 |
| **NFR-5 決定性/セーブ** | タイマーは**トランジエント状態**（Context でなく `useBattleOrchestrator` 配下）。セーブ対象外。ただし `turnTimeSec` の算出は装備/速度/深度/難易度から**決定的に再現可能**であること。 |
| **NFR-6 既存規約** | React 19 パターン（render 中 ref 参照禁止・派生 state は render-time 比較）・状態単一所有・不可侵コード（`deck.ts`/`deckReducer.ts`）厳守。 |

## 6. 持ち時間の算出モデル（初版 / 要バランス）

> 数値は**プレースホルダ**。Slice 2 以降に `card-battle-balance-auditor` 等でチューニングする。

```
turnTimeSec = clamp(
    BASE[difficulty]                       // 難易度基準値
      * enemyDepthMultiplier               // 敵/深度補正（< 1.0 で短縮）
      + playerTempoBonus                   // 速度/装備補正（秒）
      + activeTimeModifier,                // 持続型カード/バフ補正（秒）
    MIN_TURN_TIME,                         // 下限（理不尽防止）
    MAX_TURN_TIME                          // 上限（冗長防止）
)
// 即時型（FR-5）はカウントダウン中の remaining に直接 +X 秒で加算
```

**初版パラメータ（仮）**:

| 要素 | 仮値 |
| --- | --- |
| `BASE` | Easy 14s / Normal 10s / Hard 7s |
| `enemyDepthMultiplier` | 通常 ×1.0 / エリート ×0.85 / ボス ×0.8、深度ごと −0.05（depth1=1.0 … depth5=0.8）。敵個体係数も許容 |
| `playerTempoBonus` | `floor(speed / K)` 秒 + 装備「機敏」系ボーナス。`speed` は既存 `BattleStats.speed` を流用 |
| `activeTimeModifier` | 時間バフ/デバフの合算（例: +3s / −2s） |
| `MIN_TURN_TIME` / `MAX_TURN_TIME` | 4s / 20s |

「バトルごとに変動」= ①②③はバトル初期化時に確定、④はバトル中（ターン単位）で動的に増減。

## 7. アーキテクチャ影響（既存実装への接続）

> 既存: バトルは `useBattleOrchestrator` がトランジエントに統括。`useBattlePhase` がフェーズキュー（速度順）とターン管理、プレイヤーフェーズは「入力待ち」。

### 7.1 新規フック `useTurnTimer`（提案）
`useBattleOrchestrator` 配下のサブフックとして新設（CLAUDE.md のフック構成に追加）。
- 責務: 現在の `turnTimeSec` 保持、カウントダウン（rAF）、`onTimeout` コールバック発火、即時/持続の時間増減 API。
- 表示用残量は `useState`（render 中に `ref.current` を読まない＝NFR-6）。内部計測は `useRef` + rAF。

### 7.2 `useBattlePhase` / `useBattleOrchestrator` 連携
- プレイヤーフェーズ突入（入力ウィンドウ open）で**タイマー開始**。
- ターン終了（手動・タイムアウト・AP 枯渇）で**停止**。
- 敵フェーズ・フェーズ遷移中は**非計測**（FR-2）。
- `onTimeout` は既存のターン終了パス（手札破棄等）へ合流させる。

### 7.3 持ち時間算出の置き場
- 純粋関数 `calculateTurnTime(difficulty, depth, enemy, playerStats, modifiers)` を `domain/battles/calculators/` に新設（テスト容易・決定的）。
- バトル初期化（`initializeBattle` 相当）で①②③を確定。

### 7.4 バフ/デバフ・カードデータ
- バフ種に「時間」系を追加（`buff_debuff_system` / `battle-system` スキル）。
- 時間増減カードは `card-creator` スキルでデータ追加（剣士/魔術師）。

### 7.5 アニメーションのノンブロッキング化（必須前提）
常時進行（FR-2）ゆえ、**カード解決アニメがタイマー/入力を止めてはならない**。
- 既存 DOM/CSS アニメ（`useCardAnimation` 等）と PixiJS レイヤーが入力・rAF を阻害しないことを担保。
- これは **PixiJS Phase 2（バトル演出移行）と密接に連動**（`vision/plans/pixijs_phase2_battle_effects.md`）。アニメ重い場合は演出を短縮 or 並行描画へ。

### 7.6 設定・難易度
- `SettingsContext` に `TIME_BY_DIFFICULTY` 連動と（採用なら）アクセシビリティトグルを追加。

### 7.7 セーブ
- タイマーはトランジエント＝セーブ非対象（NFR-5）。

## 8. 受け入れ基準（Iteration 0 / MVP = Slice 1）

- **Given** プレイヤーフェーズ開始 **When** タイマーが起動 **Then** 残り時間がバー＋数値で表示され、固定 10s から減少を始める。
- **Given** カウントダウン進行中 **When** カードを解決/アニメ表示 **Then** タイマーは**停止しない**（常時進行）。
- **Given** 残り時間 0 **When** タイムアウト **Then** 未使用 AP・手札を放棄して**ターンが強制終了**し、敵フェーズへ正しく遷移する。
- **Given** 敵フェーズ/フェーズ遷移中 **When** 計測 **Then** プレイヤーの持ち時間は**消費されない**。
- **Given** プレイヤーが時間内に「ターン終了」 **When** 早期終了 **Then** 正常にターンが終わる。
- 既存テストが全て pass（回帰ゼロ）。

## 11. 次イテレーションで詰める未決事項（Open Questions）

## 12. リスクと対策

| リスク | 影響 | 対策 |
| --- | --- | --- |
| 常時進行 × 重いアニメ | タイマーがアニメに食われ理不尽化（最大リスク） | NFR-2 を最優先。S1 で「アニメ中も止まらない」を実体験し、S4 で PixiJS Phase 2 と非ブロッキング統合 |
| 難易度の急騰 | 既存プレイヤーにとって過酷 | 難易度連動 + アクセシビリティトグル。Easy で熟考余地を残す |
| 戦略性の喪失 | 反射ゲーム化し「本格戦略」要求と矛盾 | front-loaded strategy（§3）。事前ビルドと時間リソース化で戦略を担保 |
| 既存バトル統合の回帰 | フェーズ/ターン遷移の破壊 | `card-battle-battle-logic-validator` を commit 前ゲートに。受け入れ基準に「回帰ゼロ」 |
| トランジエント状態の React 19 落とし穴 | render 中 ref 参照・set-state-in-effect 衝突 | NFR-6 厳守。`card-battle-state-invariant-checker` で点検 |
```
