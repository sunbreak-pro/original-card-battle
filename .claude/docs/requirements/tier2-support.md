## R2-1: ギルドの噂 → 敵情報の初期ヒント（B2）

- **由来**: concept §3 B2 / 統合ギャップ（CAMP-3 + ENM-7 + JNL-6。前提整理に CAMP-4, DUN-13）/ DOC-17
- **内容**: guild 設計書（DOC-17）を改訂。ギルドで購入する噂に `enemy_intel` 型を追加し、対象敵の手記に初期ヒント1行を書き込む（モック①の手記パネル「噂: 尾の一撃は受けるな」）。所有: 噂の購入・有効状態 = GuildContext、手記への書き込み = JournalContext の公開 API。実施順: CAMP-4（噂状態の一本化）→ 本要件 → DUN-13（旧率系噂の処遇確定）
- **依存**: R1-13, R1-12
- **受け入れ基準**:
  - [ ] 噂を購入すると対象敵の手記にヒントが追加され、初遭遇から表示される
  - [ ] 噂状態の書き込み箇所が GuildContext のみ
