## 実験候補（コンセプトと整合するが今回見送った軸）

### R3-1: 身体リアリティのフル実装

- **由来**: concept §1 で「スタミナのみ先行導入」と決定した残り
- **内容**: 負傷部位 / 出血の恒常化 / 疲労のラン持ち越し / 空腹 / 治療に時間がかかる。スタミナ（R1-5）が定着した後、ラン横断の疲労として拡張する案が最有力
- **着手条件**: Tier 1-2 完了後、通しプレイで「もっと身体が欲しい」と感じた場合

### R3-2: 世界・生活リアリティ

- **由来**: concept §1 見送り軸（世界・生活）
- **内容**: 経済の需給変動 / 時間経過 / 天候 / NPC の生活。R2-2 環境ルールの先に「ステージの天候が日替わりで変わる」等の小さい入口がある
- **着手条件**: 宿場町（B2）の世界観が定着し、キャンプ滞在の動機を増やしたくなった場合

### R3-3: アナログ TCG 質感・確率の透明性

- **由来**: concept §1 見送り軸（アナログ TCG / 確率透明性・物理）
- **内容**: カードの質感・希少性演出、ダイスや判定の見える化。PixiJS 演出（R2-3）の自然な拡張先
- **着手条件**: R2-3 完了後の演出強化フェーズ

### R3-4: 3ステージ目以降の拡張

- **由来**: concept §3 ステージ構成「開発中はまず2ステージを用意」の先の拡張（3ステージ目以降）
- **内容**: StageDefinition（R1-11）にデータを足すだけで増やせる構造が前提。テーマ候補は世界観掘り下げセッションで追加検討
- **着手条件**: 2ステージの通しプレイ合格（合格基準2）後

## 凍結（新要件に含めない。廃棄もしない）

- **由来**: concept §5 スコープ境界「凍結 + PixiJS のみ統合」
- **対象**: AoE Cards / Quest System / Title System / NPC Conversation / Dark Market Expansion（各プラン書は `docs/vision/plans/` に保管継続）
- **注意**: Quest System は既に GuildProvider に一部実装済み（updateQuestProgress）。凍結 = 拡張しない・新コア適合の保証もしない。ただし R1 実装で壊れた場合は「動かないまま放置せず、無効化して隔離」する（壊れたまま見せない）
- **解凍条件**: Tier 1-2 完了後にユーザーが明示的に再優先順位付けした場合のみ

## 廃棄（明示的に作らないと決めたもの）

- 旧 Difficulty 軸の温存（§6 決定#4 で廃止確定 — 「残す」案はここに葬る）
- TexturePacker 等の有償ツール依存（$0 厳守、修正#12）
