## 1. リアル性の定義（§4.1 確定）

> **本作におけるリアル性とは、「戦闘が間合い・体勢・スタミナを持つ現実的な駆け引きであり（主柱）、その結果が不可逆に残り、敵の情報は観察によってのみ得られる」ことである。**

### 採用軸の優先順位表

| 優先      | 軸                                                            | 強度・備考                                                                                         |
| --------- | ------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| 1（本命） | 戦闘のリアリティ                                              | 主要システム化。間合い・体勢・武器の重さ。カード＝「実際に取れる行動」。設計衝突時はこの軸を最優先 |
| 1に内包   | スタミナ                                                      | 戦闘内リソース（重い行動ほど消費、戦闘終了でリセット）。身体リアリティの先行導入分                 |
| 2         | 結果の重さ（不可逆性）                                        | 死・喪失が重い。回復が安くない。主柱を支える副柱                                                   |
| 3         | 情報のリアリティ                                              | 敵ステータスは初見では非開示。観察・経験で判明。主柱を支える副柱                                   |
| 見送り    | 身体（フル実装）/ 世界・生活 / アナログTCG / 確率透明性・物理 | Tier 2/3 で再検討可                                                                                |

**強度**: 主要システム化 — 採用軸が中核メカニクスの一部を置き換え・拡張する。

## 2. コアファンタジーと体験の柱（§4.2 確定）

**コアファンタジー（1行）**:
無双の英雄ではなく、生身の体ひとつで「歪んだ土地」に潜る冒険者。一手の判断ミスが命取りになる。

**体験の柱（3本）**:

1. **間合いの読み合い** — 距離・体勢・スタミナを踏まえた手札の駆け引き（本命軸の中核体験）
2. **観察と学習** — 未知の敵を観察して情報を積み上げ、次の遭遇で活かす
3. **成長の手応え** — カード熟練度・装備・技術の積み上げによる上達感（既存の熟練度→派生システムを活用）

**トーン・世界観**: 現状のダークファンタジー（腐食/狂乱/混沌/虚無/深淵）を**ライト幻想寄りに調整**する。

## 3. 世界設定（§4.2 掘り下げ・採用分）

| ID  | 設定                                                                                                            | 接続する軸                          |
| --- | --------------------------------------------------------------------------------------------------------------- | ----------------------------------- |
| A1  | **手記は死を越える** — 観察した敵情報は手記（Journal）に蓄積。探索者が死んでも手記＝知識だけは次に引き継がれる  | 情報 × 不可逆性 × 既存 Journal 資産 |
| A2  | **魔法は希少・回復は高価** — 治癒は貴重で回復が安くない世界律。「生身の探索者」の裏付け                         | 不可逆性                            |
| B1  | **環境ルール付きの「歪みの土地」** — 各ステージは独立テーマの土地。環境（視界・足場・広さ）が間合い・体勢に影響 | 戦闘（本命）                        |
| B2  | **キャンプ＝宿場町** — 既存5施設をダンジョン群の中継地に再解釈。ギルドの噂が敵情報の初期ヒント                  | 情報 × 既存キャンプ資産             |
| B3  | **難易度＝歪みの濃さ** — ギルドが各ステージの推奨等級を掲示。どこからでも入れるが難易度が違う                   | ステージ構成の世界観裏付け          |

不採用: A3（敵の生態デザイン）→ 敵デザインは現状ベースを維持。

### ステージ構成

- 現行「1ダンジョン×5深度×5フロア」から「**複数ダンジョン（ステージ）× テーマ別の複数階層**」へ変更
- 開発中はまず **2ステージ** を用意（候補例: 翠苔の森殿＝初級 / 霧の水郷＝中級）
- 基本的にどのステージからでも開始可。ステージ選択＝難易度選択になる構成

## 4. 既存資産との関係（§4.3 確定）

**方針: B — コア再設計・資産流用**

- 戦闘コア（間合い・体勢・スタミナ）とダンジョン構造（複数ステージ制）は設計から見直す
- データ定義・UI・PixiJS 基盤・キャンプ実装・熟練度システム・Journal は流用
- ラン制ローグライク＋キャンプのループ自体は維持

**クラス**: まず**剣士のみ**で新戦闘コアを再設計・完成させる。魔術師（属性共鳴）は後追いで適合。
