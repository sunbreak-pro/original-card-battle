カードの読み方（新カードフレーム）

カード＝「実際に取れる行動」。1枚に「体力で払うコスト」「届く距離」「体への影響」を載せる。

3
大上段斬り
⚔️
ダメージ 18。重い一振り。当たれば敵の体勢を大きく削る。
体勢 −大（外すと崩れる）
左上の円 = スタミナコスト
従来の AP に代わる戦闘内リソース。重い武器・大技ほど高い。毎ターン少量回復し、戦闘終了でリセット。
右上の 3 ドット = 間合い適性（遠・中・近）
点灯している間合いでのみプレイ可能。今の間合いで使えないカードは手札で灰色になり、理由が表示される。
下段の帯 = 体勢への影響
大技は自分の体勢を崩すリスクを持つ。体勢が崩れる（メーターが尽きる）と一定時間、被ダメージ増・回避不能。敵にも同じルールが適用され、「崩してから叩く」が成立する。
数値非表示の思想
敵側の数値が見えないぶん、自分側のコスト・距離・リスクは完全開示する。「読めないのは敵、ブレないのは自分の腕」。

既存カード型（cardTypes）に「スタミナコスト / 間合い条件 / 体勢影響」の3属性を足すイメージ。熟練度→派生システムはそのまま「成長の手応え」として継続。

観察と手記 — 敵情報の段階開示

敵のステータスは隠す代わりに、「観察するほど読める」進行を 4 段階で設計する。観察は戦闘中の行動（見切り等）と遭遇回数で進む。

観察 ☆☆☆（初遭遇）

  • 名前・見た目
  • ギルドの噂（あれば）
  • HP ???
  • 意図 ???
  • 弱点 ???

観察 ★☆☆

  • HP の手応え（多い/少ない）
  • 行動の癖 1 つ
  • 意図 ???
  • 弱点 ???

観察 ★★☆

  • 次の行動の意図が見える
  • おおよその攻撃力
  • 弱点 ???

観察 ★★★（完全）

  • HP 数値開示
  • 全行動パターン
  • 弱点・耐性

間合い表現の A/B 比較

間合いは本命軸の中心 UI。画面の見せ方として 2 案を用意した。

案A: 間合いトラック型（①の画面で採用）

🗡️ 🦎
遠間
中間
近間
🗡 🦎

長所: 位置関係が常時見える。移動カードの効果が直感的。複数敵でも敵ごとのトークンで拡張できる。PixiJS でキャラ位置を実際に動かす演出（Phase 2-4 統合先）と相性がよい。
短所: 画面中央に常設 UI が増える。複数敵で間合いが敵ごとに違うと情報量が上がる。

案B: 間合いバッジ型（ミニマル）

🗡️ 🦎
⇔ 中間

長所: 実装が軽く、現行画面をほぼ崩さない。状態は1つのバッジ＋手札のグレーアウトで伝える。複数敵はバッジを敵フレーム側に付ければよい。
短所: 「詰める/離す」の操作感が薄く、本命軸の体感が弱まる。キャラが動かないので位置の納得感が出にくい。

案A トラック型案B バッジ型
本命軸（間合いの読み合い）の体感◎ 位置が動く・読み合いが見える△ 状態表示に留まる
実装コスト中〜大（位置状態 + UI + 演出）小
PixiJS Phase 2-4 との接続◎ キャラ移動演出の受け皿になる○ 演出は別途設計
複数敵対応○ トークン拡張（設計必要）○ 敵フレームにバッジ
段階導入B から A へ育てられるA の先行ステップにできる

推奨: 最終形は案A（本命軸の体感を最大化、PixiJS 統合先としても自然）。ただし実装は「案B → 案A」の2段階で進めると、グレーアウト・間合い状態などのロジック層を先に固めてから視覚層に投資できる。
